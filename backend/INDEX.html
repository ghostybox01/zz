🎯 Quick Start (3 Steps)

1 Edit Configuration Files
  • server_ips.txt - Add your VPS IP addresses (one per line)
  • targets.txt - Add URLs to scan (one per line)
  • ssh_config.json - Optional, defaults are fine
2 Run ONE Command

Windows:

Double-click: START.bat

Or from Command Prompt:
python start.py

Linux/macOS:

chmod +x START.sh
./START.sh

Or:
python3 start.py
3 Access Dashboard
  • Dashboard: http://localhost:5000
  • VPS Control: http://localhost:5000/vps

⚡ What Happens Automatically

  • ✅ Checks if Go is installed (auto-installs on Linux)
  • ✅ Downloads Go dependencies
  • ✅ Builds raven-scanner binary
  • ✅ Creates deployment package
  • ✅ Tests SSH connections to all VPS
  • ✅ Uploads scanner to all VPS
  • ✅ Splits targets equally across VPS
  • ✅ Starts scanning on all VPS
  • ✅ Launches real-time dashboard

🎮 Command Options

# Full automation (default)
python start.py

# Skip deployment, just build and run dashboard
python start.py --no-deploy

# Only run dashboard (skip build and deploy)
python start.py --dashboard-only

# Force rebuild even if scanner exists
python start.py --rebuild

# Show help
python start.py --help

⚠️ Prerequisites

  • Python 3.7+ - Must be installed
  • Go 1.21+ - Auto-installs on Linux, manual on Windows
  • SSH Key - At path specified in ssh_config.json
  • VPS Servers - With root access

Windows: Install Go

If "Go not found" error appears:

  1. Download: go1.21.6.windows-amd64.msi
  2. Install and restart terminal
  3. Run START.bat again

🔐 SSH Key Setup (If Needed)

If you don't have SSH key yet:

# Generate SSH key
ssh-keygen -t rsa -b 4096 -f /root/ssh/1

# Copy to each VPS
ssh-copy-id -i /root/ssh/1.pub root@66.228.42.201
ssh-copy-id -i /root/ssh/1.pub root@66.228.42.244

🛠️ Troubleshooting

Problem: "Go not found" on Windows

Fix: Install Go from go.dev/dl/ and restart terminal

Problem: "SSH connection failed"

Fix:

# Test manual SSH
ssh -i /root/ssh/1 root@

# Check key permissions
chmod 600 /root/ssh/1

# Verify config
cat ssh_config.json

Problem: "No servers reachable"

Fix:

  • Check server_ips.txt has correct IPs
  • Verify SSH key copied to servers
  • Check firewall allows port 22

Problem: "targets.txt not found"

Fix:

echo "https://example.com/config.php" > targets.txt
